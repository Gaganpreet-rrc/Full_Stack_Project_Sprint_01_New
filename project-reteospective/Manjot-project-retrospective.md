# When did you feel that your team's work on the project was most effective or efficient? Are there any specific practices or methods that you or your team were following that you think contributed to things going well?
- We started using feature-based branches and pull requests, which improved organization.
- The Kanban board helped track tasks clearly and made the sprint more structured.
- Team communication improved, which helped solve problems faster together.
- Instructor feedback from earlier sprints helped identify mistakes early and improve the workflow.

# When did you feel that your team's work on the project was least effective or efficient? Are there any specific problems your team encountered or ineffective practices that your team used that you think contributed? 
- In the beginning, proper branching was not used, which caused confusion and affected marks.
- Merge conflicts occurred often due to poor Git practices and large file changes.
- CORS errors and API routing issues created delays during development.
- Clerk authentication setup was difficult at first but became clearer later with practice.

# What experiences contributing to this project do you think you can learn from for future work in this field? Are there anything practices that you want to follow or develop upon? Are there any that you want to avoid, or problems that you want to address?
- Learned the importance of teamwork by collaborating to fix issues like API errors and Clerk authentication problems.
- Understood the need to adjust timelines when tasks take longer than expected during development.
- Using a backlog from the beginning helps in organizing tasks and planning work properly.
- Clear division of work between frontend and backend improves efficiency and reduces confusion.
- Improving debugging skills and understanding full-stack flow helps in solving issues faster and avoiding delays in integration.
