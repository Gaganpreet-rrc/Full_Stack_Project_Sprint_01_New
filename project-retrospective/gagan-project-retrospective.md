# When did you feel that your team's work on the project was most effective or efficient? Are there any specific practices or methods that you or your team were following that you think contributed to things going well?
- From Sprint 3, our team became more effective as we had a better understanding of Git features like creating branches for each feature and merging them properly after getting approval from the reviewer, along with a better understanding of how to manage a Kanban board to organize and track tasks more effectively. 
- We followed feedback from sprint 1 and 2 provided by our instructor, which helped us correct mistakes early and improve our workflow.  
- We also collaborated more and researched together, which helped us solve problems faster and work more efficiently.
- We used specific methods like feature-based branching, pull requests for code review, and regular commits to manage our work efficiently and keep the project organized.


# When did you feel that your team's work on the project was least effective or efficient? Are there any specific problems your team encountered or ineffective practices that your team used that you think contributed?
- In the beginning, our work was less effective because we were not creating separate branches for each feature and were working on everything together.  
- This caused confusion, merge conflicts, and made it harder to track changes properly.  
- We also faced Vercel deployment limitations where we had to wait around 24 hours or manage limited deployment attempts, which delayed testing and slowed down our progress.
- A specific problem we faced was API and authentication issues, such as incorrect routes, CORS errors, and Clerk integration problems, which required extra debugging time.  
- We also faced issues when switching branches, where files were sometimes overwritten or lost.
- There was confusion in understanding Clerk authentication since it was our first time using it.


# What experiences contributing to this project do you think you can learn from for future work in this field? Are there anything practices that you want to follow or develop upon? Are there any that you want to avoid, or problems that you want to address?
- From this project, I learned the importance of strong teamwork by collaborating on debugging issues like API errors and Clerk authentication together and adjusting timelines when tasks took longer than expected.  
- I want to continue using practices like creating a backlog at the start, dividing tasks clearly (frontend/backend), and using Git branches and pull requests for organized development.
- In the future, I will focus on improving my debugging skills and understanding full-stack flow better so I can solve issues faster and avoid delays during integration.
