# When did you feel that your team's work on the project was most effective or efficient? Are there any specific practices or methods that you or your team were following that you think contributed to things going well?
- From Sprint 3, our team was most effective after applying the feedback from Sprint 1, which improved our overall workflow.
- We followed specific practices like organizing the Kanban board properly, which helped us clearly track tasks and understand progress.
- We used detailed pull request descriptions as a method to make code reviews easier and clearer for the team.
- We followed feature-based branching, where we created separate branches for each task to reduce merge conflicts and keep work structured.
- Assigning pull requests to specific team members helped improve accountability and ensured tasks were completed on time.
- We also followed sprint planning methods by adding proper iterations, start and end dates, and setting priorities, which helped us stay focused and work more efficiently.

# When did you feel that your team's work on the project was least effective or efficient? Are there any specific problems your team encountered or ineffective practices that your team used that you think contributed?
-  At the start of the project, our team was less efficient because we did not use separate branches and worked on the same project files at the same time.
- This led to confusion, frequent merge conflicts, and difficulty in tracking changes.
- We had problems while switching branches (checkout), where some changes were accidentally overwritten or lost.

# What experiences contributing to this project do you think you can learn from for future work in this field? Are there anything practices that you want to follow or develop upon? Are there any that you want to avoid, or problems that you want to address?
- I learned that planning tasks and organizing work by using a Kanban board is very important for managing projects effectively.
- I want to continue using best practices like creating separate branches, writing clear pull requests, and committing code regularly.
- I learned the importance of understanding tools like Clerk and Prisma early to avoid confusion later.
- I will focus on testing features early and debugging step by step instead of waiting until the end.
- I want to improve communication with the team to avoid misunderstandings and delays.
- I will avoid working on the same code without proper structure and make sure to follow a clear workflow to reduce errors and conflicts.
